# GitHub 仓库管理工具

批量管理 GitHub 仓库的桌面工具，支持克隆、上传、更新操作。

## 功能

### 仓库管理
- 获取云端仓库列表（显示私有/公开状态、更新时间）
- 新建仓库（从云端克隆到本地）
- 新建云端仓库（在GitHub上创建新仓库）
- 修改仓库可见性（公开/私有切换）
- 重命名仓库（本地目录+云端仓库同时改名）

### 同步操作
- 更新（Pull）- 检查本地vs云端谁更新，有冲突时提示确认
- 上传（Push）- 自动提交并推送，云端有更新时警告
- 初始化Git - 对未初始化的目录执行 `git init`

### 其他功能
- Token验证 - 验证GitHub Personal Access Token有效性
- 双击打开 - 双击本地仓库直接打开文件夹，双击云端仓库打开GitHub页面
- 焦点刷新 - 切换窗口时自动刷新本地仓库状态
- 实时日志 - 所有操作记录带时间戳

## 安装

```bash
pip install -r requirements.txt
```

或手动安装：

```bash
pip install requests>=2.28.0
```

## 运行

```bash
python github_manager.py
```

## 配置

首次运行需填写：
- GitHub 用户名
- Personal Access Token
- 仓库存放目录

配置自动保存到 `.env` 文件，下次启动自动加载。

## 界面说明

| 区域 | 说明 |
|------|------|
| 设置区 | 仓库目录、用户名、Token配置 |
| 本地仓库 | 显示本地仓库列表及状态（已同步/有修改/未初始化） |
| 云端仓库 | 显示GitHub上的仓库列表（需先验证Token） |
| 操作按钮 | 刷新、下载、上传、新建等操作 |
| 日志区 | 实时显示操作日志 |

## 注意事项

1. **Git路径**：默认使用 `C:\Program Files\Git\cmd\git.exe`，如需修改请编辑代码
2. **Token权限**：Token需要 `repo` 权限才能创建仓库和修改可见性
3. **网络要求**：操作GitHub需要网络连接
4. **超时设置**：克隆操作超时时间为600秒，其他操作为300秒
